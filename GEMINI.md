# Umaxica App (JIT)

## Project Overview

This is a Ruby on Rails application, likely the backend for a larger system named "Umaxica." It uses
a modern technology stack, including Ruby 4.0, PostgreSQL 18+, and a JavaScript/TypeScript frontend
managed with pnpm.

The application is architecturally complex, featuring:

- **Modular Design:** The codebase is organized into various components, as seen in the `app/`
  directory structure and the modular routing configuration.
- **Multiple Services:** It integrates with a wide range of services, including AWS (SNS, Connect,
  Polly, SES), Cloudflare (R2), Fastly, Google Cloud (Cloud Run, Cloud Build, etc.), Stripe for
  billing, and Twilio for telecommunications.
- **Background Jobs:** It uses Solid Queue for processing background jobs.
- **Security Focus:** The project incorporates several security-related gems and tools, such as
  `rack-attack` for DoS protection, `pundit` for authorization, WebAuthn for authentication, and
  `brakeman` for static analysis.
- **Modern Frontend:** The frontend is built with Hotwire (Turbo and Stimulus) and styled with
  Tailwind CSS. It uses Biome for code quality.

## Building and Running

### Prerequisites

- Ruby 4.0+
- pnpm 10.x + Node 20+
- Docker (with PostgreSQL 18+, Valkey, and Kafka)

### Initial Setup

1.  Start the Docker services: `docker compose up`
2.  Install Ruby dependencies: `bundle install`
3.  Install JavaScript dependencies: `pnpm install`
4.  Prepare the database: `bin/rails db:prepare`
5.  Run the development server: `bin/dev`

### Testing

- Run the full test suite: `bundle exec rails test`
- Run tests with coverage: `COVERAGE=true bin/rails test`

### Linting & Formatting

- **Ruby:** `bundle exec rubocop`
- **ERB:** `bundle exec erb_lint .`
- **JavaScript/TypeScript:** `pnpm run check`

## Development Conventions

- **Logging:** Use `Rails.event.record("event.name", payload_hash)` or `Rails.event.error(...)` for
  structured logging.
- **Database IDs:** Primary keys are UUIDv7 generated by PostgreSQL's native `uuidv7()` function.
- **Secrets:** Sensitive configuration is stored in Rails credentials.
- **Pre-commit Hooks:** Lefthook is used for pre-commit automation, which includes running
  `git-secrets --scan`.
- **Type Checking:** The project uses Sorbet for static type checking in Ruby.
- **Code Style:** Adhere to the configurations for RuboCop (Ruby) and Biome (JavaScript/TypeScript).
